
# Anti-patterns: UX flow auditor

- NEVER store flow reports outside `.heyeddi/docs/ux-flows/`
- NEVER confuse with `@heyeddi-design critique` (static heuristics) or `@visual-auditor` (pixels)
- NEVER trace without dev server running
- NEVER set unrealistic `max_clicks`: derive from product acceptance criteria
- NEVER skip friction notes: metrics alone do not explain fixes
- NEVER ship AI prose slop (em/en dashes, delve/leverage/tapestry, "Certainly!", "it is important to note", emoji theater); follow `context/PROSE_ANTI_SLOP.md` fully
